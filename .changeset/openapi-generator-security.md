The OpenAPI reference is now generated by Futonic itself rather than better-call's built-in generator, producing a spec-accurate document: every method on a shared path is kept (no more dropped `POST` when a `GET` is registered on the same path), all HTTP verbs are emitted, and `:id` route segments are templated as `{id}` path parameters. Authentication is document-driven and auth-agnostic — declare `securitySchemes` and a document-wide `security` requirement via the `openApi` handler option; no scheme is asserted by default. New exports: `generateOpenApiDocument`, `openApiReferenceHtml`, and `OpenApiOptions`/`SecurityScheme`/`SecurityRequirement`.
